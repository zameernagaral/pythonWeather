🎓 Lab Evaluation & Code Tour Guide (Click to expand notes for your teacher)

This application was designed to showcase clean, structured, and modular Python programming. Here is an overview of how the app functions to help you explain it to your lab instructor:

1. Request Flow (How it Works)

  1. The user inputs a city name and submits the form (HTML).
  2. The form is sent as a `POST` request to the Flask server (`app.py`).
  3. The backend converts this to a `GET` request with query params (`?city=Name&units=c`) for clean URL routing.
  4. Flask triggers the `index()` view function, extracts the parameters, and queries the OpenWeatherMap API using Python's standard `urllib` library.
  5. The API returns a JSON payload which Python's standard `json` module parses into a native Python dictionary.
  6. Python logic computes unit conversions, runs the recommendation conditional checks, updates the search history list, and passes all variables to the Jinja2 engine, which compiles the styled HTML page.

2. Key Python Concepts Used

  • Libraries: `urllib.request` (making HTTP queries without external libraries), `json` (parsing strings into dictionary structures), and `flask` (web framework, sessions, and HTML templates).
  • Dictionary Parsing: Extracting nested elements using safe `.get()` calls (e.g. `data.get("wind", {}).get("speed")`) to prevent server crashes if the API misses some data.
  • Flask Sessions: Utilizes browser cookie sessions (`session["history"]`) to persist recent searches client-side. Flask automatically encrypts this list using a server-defined secret key.
  • Conditional Logic: If-elif-else statements used in `generate_recommendations()` to inspect metrics (temperatures, wind speed, and weather descriptions) and dynamically generate localized advice.
